# 04 リスク管理 — 米株デイトレード知識ベース

> 対象: IBKR 米株デイトレードボット（Ouroboros）のリスク設計改善
> 作成: 2026-06-11 / 出典は末尾の「参考ソース」を参照（数式は原典引用）

---

## 結論先出し（要旨）

1. **リスク管理の核は「破産しないこと」**。期待値がプラスでも、1トレードのリスクが大きすぎると連敗で口座が消える（risk of ruin）。
2. **SLはボラティリティ（ATR）基準で置く**。固定%のSLは、銘柄のノイズ幅より狭いと「正しい方向でも狩られる」。現ボットの -0.5%固定SLは AMD（ATR≈2.055 / price≈455）で **ATRの約11%幅** しかなく、明らかに狭すぎる（後述§9で計算）。
3. **R:Rと勝率は損益分岐でつながる**。R:R 2:1 の損益分岐勝率は 33.3%。だが「狭いSL→TPに届く前に狩られる」と実効R:Rが崩れ、分岐勝率を割り込む。
4. **ポジションサイズは口座%固定（1〜2%）＋連敗時に縮小／休止**。ケリーは上限の目安に留め、実運用はハーフケリー以下。

---

## 1. ポジションサイジング

### 1.1 固定比率法（Fixed Fractional / 2%ルール）

最も基本的かつ推奨される方式。**1トレードで失う額を口座残高の固定%に固定する**。残高が増減すると賭け額も自動で連動する（複利・自己縮小）。

```
ポジションサイズ（株数） = (口座残高 × リスク%) ÷ (1株あたりのSL幅 $)
```

- **2%ルール**: 1トレードのリスクを口座の 2% 以内に抑える。CFA Institute も単一トレードのリスクを総資本の 2% 以下に推奨（出典: arongroups / CFA引用）。
- **1%ルール**: より保守的。初心者・検証中の戦略は 0.5〜1% が安全（出典: daytradingtoolkit / backtestbase）。
- リスク%と想定ドローダウンの関係（95パーセンタイル最悪DD、出典: thearcalabs）:

| 1トレードリスク% | 95%タイル最悪DD（目安） |
|----------------|----------------------|
| 1% | 約 18% |
| 2% | 約 32% |
| 5% | 60%超 |

> ポイント: リスク%は「勝率」ではなく「生存」を決める変数。DDが大きいほど精神的・複利的に回復が困難。

### 1.2 口座に対する%（ポジション総額 vs リスク額の区別）

混同しやすい2つを分ける:
- **ポジション総額%**: いくら建てるか（例: 口座の50%でQQQを買う）。
- **リスク額%**: 負けたらいくら失うか（= ポジション総額 × SL幅%）。**リスク管理で固定すべきはこちら**。

例: 口座$1,000、QQQ $500建玉、SL -1% → リスク額は $5（口座の0.5%）。建玉が口座の50%でも、SLが浅ければリスクは小さい。

## 2. リスクリワード比（R:R）と期待値（Expectancy）

### 2.1 R:R 定義

```
R:R = 平均利益（リワード） ÷ 平均損失（リスク）
```
SL幅とTP幅で事前に決まる。TP +1.0% / SL -0.5% なら R:R = 2.0（2:1）。

### 2.2 期待値（Expectancy）— 原典の式

出典: traderssecondbrain / luxalgo

```
Expectancy = (勝率 × 平均利益) − (敗率 × 平均損失)
```

R倍数（R-multiple）表記（平均損失を1Rに正規化）:
```
Expectancy(R) = (勝率 × R) − (敗率 × 1)
   ここで R = 平均利益 ÷ 平均損失、 敗率 = 1 − 勝率
```

**1トレードあたり何R稼げるか**を表す。プラスなら長期的に増える。

### 2.3 損益分岐勝率（Break-even Win Rate）— 原典の式

出典: pnlledger / journalplus

```
損益分岐勝率 = リスク ÷ (リスク + リワード) = 1 ÷ (1 + R:R)
```

| R:R | 損益分岐勝率 | 計算 |
|-----|------------|------|
| 1:1 | 50.0% | 1 ÷ (1+1) |
| 1.5:1 | 40.0% | 1 ÷ (1+1.5) |
| **2:1** | **33.3%** | 1 ÷ (1+2) |
| 2.5:1 | 28.6% | 1 ÷ (1+2.5) |
| 3:1 | 25.0% | 1 ÷ (1+3) |
| 4:1 | 20.0% | 1 ÷ (1+4) |

> 勝率とR:Rは反比例（出典: luxalgo）。TPを広げてR:Rを上げると、TP到達確率が下がり勝率は落ちる。**「広いTPの裏で狭すぎるSLを置く」と最悪**: 勝率は下がるのにSLは簡単に狩られ、実効R:Rも崩れる。

### 2.4 具体計算: 勝率40% / R:R 2:1

平均損失を 1R（=1単位）とする。
```
Expectancy(R) = (0.40 × 2) − (0.60 × 1)
              = 0.80 − 0.60
              = +0.20R / トレード
```
→ 1トレードあたり平均 **+0.20R** のプラス期待値。$10リスクなら平均 +$2/トレード。
損益分岐勝率33.3%に対し実勝率40%なので **+6.7pt の余裕**。健全。

## 3. ケリー基準（Kelly Criterion）

### 3.1 公式（原典）

出典: zerodha varsity / enlightenedstocktrading / traderspost

```
Kelly% = 勝率 − (敗率 ÷ R:R)
       = W − (1 − W) / R
   W = 勝率（小数）、 R = ペイオフ比（平均利益 ÷ 平均損失）
```

**意味**: 長期の資本成長率（対数成長）を最大化する、1トレードあたりの最適賭け比率。

### 3.2 計算例（勝率40% / R:R 2:1）

```
Kelly% = 0.40 − (0.60 / 2.0) = 0.40 − 0.30 = 0.10 = 10%
```
→ フルケリーは口座の10%をリスクに。**だがこれは過大**。

### 3.3 ハーフケリー・保守運用の理由

- **フルケリーはボラティリティが極端に高い**（DDが深い）。Ed Thorp（ケリーを金融に持ち込んだ数学者）は **ハーフケリー以下**を推奨（出典: tradesearcher / journalplus）。
- **ハーフケリー（50%）**: 最適成長率の約75%を維持しつつ、分散（ばらつき）を約半分に。DDが大幅に小さく、エクイティカーブが滑らか。
- **クォーターケリー（25%）**: 成長率の約50%だがさらに低ボラ。

| 方式 | リスク%（上の例） | 成長率維持 | 特徴 |
|------|----------------|-----------|------|
| フルケリー | 10% | 100% | DD過大・実用外 |
| ハーフケリー | 5% | 約75% | 実務標準 |
| クォーターケリー | 2.5% | 約50% | 保守 |

> 実務指針: ケリーは**サイズの上限の目安**。勝率・R:Rの推定誤差があるため、推定がブレるほど実額は小さく（ハーフ以下に）倒す。§1の2%ルールはほぼクォーターケリー相当の保守域。

## 4. 損切り設計: 固定% vs ATRベース

### 4.1 ATR（Average True Range）とは

直近N本（通常14）の値動き幅の平均。銘柄ごとの「1本あたりの普通の揺れ幅（ノイズ）」を表す絶対値（$）。

### 4.2 優劣比較（出典: quantstock / tradingview Capitalcom / chartswatcher）

| 観点 | 固定%SL | ATRベースSL |
|------|---------|-----------|
| ボラ適応 | しない（全銘柄同じ%） | する（銘柄ごと自動調整） |
| ノイズ耐性 | 低い（狭いと狩られる） | 高い（ノイズ幅外に置ける） |
| 高ボラ銘柄 | 危険（即狩られ） | 自動で広がる |
| 低ボラ銘柄 | 緩すぎる場合あり | 自動で締まる |
| 弱点 | 銘柄特性を無視 | レンジ・乱高下相場では逆に過敏。サポレジ併用推奨 |

**結論**: デイトレでは **ATRベースSLが基本的に優位**。固定%は「銘柄の素性を無視する」のが致命的欠陥。

### 4.3 ATRベースSLの式

```
SL幅($) = ATR × 倍率k    （k は通常 1.5〜3.0）
SLライン = エントリー価格 − (ATR × k)   ※ロングの場合
```

### 4.4 タイト過ぎるSLの問題（ノイズ狩り）

- SL幅が ATR より狭いと、**方向が合っていても通常の揺れ（ノイズ）でSLにヒット**する（whipsaw / 出典: chartswatcher / quantstock）。
- 「正しいトレードなのに損切りが量産される」典型パターン。**まさに現ボットのSELL7連敗の疑い**（§9）。
- 逆にATRが極小のレンジ相場ではATRストップも近すぎになりうる → サポレジ等の市場構造と併用（出典: tradingview）。

## 5. 日次損失上限・連敗ストップ・クールダウン

| 仕組み | 目的 | 一般的な設定例 |
|--------|------|--------------|
| 日次損失上限（daily loss limit） | 1日の傷を限定。悪い日を早期撤退 | 口座の 2〜6% / 固定$ |
| 最大連敗ストップ（streak stop） | 環境不一致・自分の崩れを検知して止める | 連続2〜3敗で当日停止 |
| クールダウン | 連敗後の「取り返し（リベンジ）取引」を防ぐ | 一定時間/当日エントリー禁止 |

考え方:
- 連敗は「相場環境が戦略と合っていない」シグナルであることが多い。**止めること自体がエッジ**。
- 損失上限ヒット後の続行は、心理崩壊→サイズ拡大→破産の典型経路。機械的に止める。

---

## 6. 最大ドローダウン（Max DD）と破産確率（Risk of Ruin）

### 6.1 破産確率の式（原典: Perry Kaufman, "Trading Systems and Methods"）

出典: journalplus / backtestbase / 上記書籍引用

```
RoR = ((1 − Edge) / (1 + Edge)) ^ Capital_Units

  Edge = 1トレードあたりの優位性
       = (勝率×平均利益 − 敗率×平均損失) ÷ 平均損失   （= 期待値R）
  Capital_Units = 破産点までのリスク単位数
                = (口座残高 × 許容DD%) ÷ 1トレードのリスク額
```

### 6.2 直感

- **Edgeが小さく、Capital_Unitsが少ない（=1トレードのリスクが大きい）ほどRoRは跳ね上がる**。
- 例: Edge=0.20、破産点まで20単位なら RoR = (0.8/1.2)^20 ≈ 0.00030（0.03%）。1トレードのリスクを倍にして10単位にすると (0.8/1.2)^10 ≈ 0.017（1.7%）と約57倍に悪化。
- 実務目安（出典: backtestbase）: 正の期待値があれば 2%/トレードで RoR はおおむね 5〜7%未満、初心者は 0.5〜1% が安全。

### 6.3 ドローダウンの非対称性

DDからの回復は非対称（必要リターンは加速度的に増える）:

| DD | 回復に必要なリターン |
|----|--------------------|
| -10% | +11.1% |
| -20% | +25.0% |
| -33% | +50.0% |
| -50% | +100% |

> だから「勝つ」より先に「大きく負けない」。Max DDを設計段階で上限化する。

## 7. ボラティリティ（VIX / ATR）に応じたサイズ調整

考え方: **「同じ$リスクを保つ」=「ボラが上がったらサイズを下げる」**。

- **ATRでサイズを正規化**:
  ```
  株数 = (口座 × リスク%) ÷ (ATR × k)
  ```
  ATRが大きい銘柄/局面ほど自動的に株数が減り、$リスクが一定に保たれる。
- **VIXゲート**: VIX（市場全体の予想変動率）が高い局面はノイズもギャップも大きい。閾値超で **サイズ縮小 or エントリー停止**。現ボットの `VIX≥30 ブロック` はこの思想（出典: 一般的実務）。
- 原則: 高ボラ＝チャンスでもあるが、固定サイズのまま突っ込むと$リスクが膨張する。**サイズで吸収**する。

## 8. 主要数式まとめ（早見表）

| 項目 | 式 |
|------|----|
| ポジション株数 | (口座 × リスク%) ÷ SL幅$ |
| 期待値(R) | 勝率×R − 敗率×1 |
| 損益分岐勝率 | 1 ÷ (1 + R:R) |
| ケリー% | 勝率 − 敗率/R:R |
| ATR-SL幅 | ATR × k (k=1.5〜3) |
| 破産確率 | ((1−Edge)/(1+Edge))^Capital_Units |
| DD回復率 | 1/(1−DD) − 1 |

## 9. このボットへの紐付け（IBKR bot 専用・最重要）

### 現状（IBKR_CONTROL.csv 実値・2026-06-11確認）

| 項目 | 値 |
|------|----|
| `ibkr_sl_pct` | **-0.5%（固定）** |
| `ibkr_tp_pct` | +1.0%（固定） |
| 名目R:R | 2:1 |
| `ibkr_shares` | 1（株数固定） |
| `ibkr_daily_loss_limit_usd` | -$20 |
| `ibkr_streak_stop_max_losses` | 2 |
| `ibkr_vix_block_threshold` | 30 |
| `ibkr_atr_tp_multiplier` | 1.5（**TPのみ**にATR適用。コード行 866–872） |

**重要な構造的欠陥**: `ibkr_atr_tp_multiplier` で **TP側はATR適応済み**だが、**SL側はATR適応が一切ない（-0.5%固定のまま）**。TPだけ広げてSLは銘柄ノイズより狭い、という最悪のアンバランス。これが7連敗の温床。

---

### ① -0.5%固定SLがATR比でどれだけ狭いか（AMD実例で計算）

前提: AMD ATR = 2.055、price ≈ 455。

```
ATRをドル幅でなく価格%に換算:
  ATR(%) = 2.055 / 455 × 100 = 0.4516%   （AMDの1本あたり通常ノイズ ≒ 0.45%）

現SL幅 = 0.5%（≒ $2.275 @455）

SL幅 ÷ ATR(%) = 0.5 / 0.4516 = 1.11倍
```

**結論**: 現SLは ATR の **わずか1.11倍**しかない。
- ATRは「1本あたりの平均的な揺れ」。SLがATRの約1倍ということは、**たった1本分の通常ノイズでSLに到達する**。
- デイトレの一般的なATRストップ（k=1.5〜3）から見て **明確に狭すぎる**。
- 「方向が合っていても1本のノイズで狩られる」状態 → SELL7連敗が全て -0.5%ストップという実績と完全に整合。**ノイズ狩りの疑いは濃厚**。

> $幅で見ても: SL $2.275 < ATR $2.055 の1.1倍。AMDが普通に1本動くだけでヒットする距離。

---

### ② ATRベースSL（ATR×1.5）に変えた場合のSL幅とR:R維持に必要なTP

ATRストップ k=1.5 を採用:
```
新SL幅(%) = ATR(%) × 1.5 = 0.4516 × 1.5 = 0.677%   （≒ $3.08 @455）
```

R:R 2:1 を維持するなら:
```
必要TP幅(%) = 新SL幅 × 2.0 = 0.677 × 2 = 1.354%   （≒ $6.16 @455）
```

| 項目 | 現行（固定） | ATR×1.5案（AMD） |
|------|------------|-----------------|
| SL幅 | 0.5%（$2.28） | 0.677%（$3.08） |
| SL ÷ ATR | 1.11倍 | 1.50倍 |
| R:R 2:1のTP | 1.0% | 1.354%（$6.16） |

補足:
- 既存の `ibkr_atr_tp_multiplier=1.5` は **TP = ATR×1.5 = 0.677%** を計算するが、コード866–872は「**effective_tp より大きいときだけ上書き**」かつ現tp_pct=1.0%の方が大きいので **実質ほぼ発火していない**。SL側に同じATR連動を入れるのが本筋。
- 提案: `ibkr_sl_atr_multiplier`（新規, 例1.5）を追加し、SLを `min(固定%, ATR×k)` ではなく **`max(固定%, ATR×k)`（=広い方を採用）** で置く。ノイズ幅を必ず外に出す。TPは `SL幅 × 目標R:R` で連動算出し、R:Rを2:1で固定する。

---

### ③ この戦略の損益分岐勝率

R:R 2:1 を維持する限り:
```
損益分岐勝率 = 1 ÷ (1 + 2) = 33.3%
```
- 名目では33.3%超えれば黒字。**だが現状はSLが狭すぎて実効R:Rが2:1で機能していない**（TP到達前にSL狩り→実質的に勝率が分岐を割る）。
- ATR×1.5でSLを正しく置けば「TPに届くまで生き残る」確率が上がり、**実効勝率が名目に近づく**。33.3%の分岐を現実的に超えやすくなる。
- 注意: SLを広げるとTPも広げる（1.354%）ため、**TP到達までの時間が延びる**。デイトレの保有時間制約と要バランス。届かないなら R:R を 1.5:1（分岐40%）に下げる選択肢も検討。

---

### ④ 連敗時のサイズ／休止ルール（数値提案）

現状 `ibkr_shares=1` 固定・`streak_stop=2`・`daily_loss=-$20`。これに段階的縮小を追加:

| 状態 | アクション |
|------|-----------|
| 通常 | 基準サイズ（1株 / または口座1%リスク） |
| 連続2敗 | **当日エントリー停止**（既存streak_stop=2を維持） |
| 連続2敗後の翌日 | サイズ **50%（ハーフケリー的縮小）**、勝てたら基準に復帰 |
| 日次 -$20到達 | 当日即停止（既存） |
| 同一銘柄で連続SL狩り（例2回） | その銘柄を当日除外＋ATR×kでSL再点検 |

クールダウン（リベンジ防止）:
- 連敗ストップ発動後は **当日再開しない**（時間クールダウンより当日終了が安全）。
- 再開前チェック: 「SL幅 ÷ ATR ≥ 1.5 か」を必ず確認してから建てる。

> サイジング思想: §3のハーフケリーに合わせ、調子が悪い局面はサイズを能動的に半減。「負けが込んだら小さく、勝ち続けたら基準」。固定1株のままだと、根本のSL狭さが直らない限り連敗が止まらない。**最優先は①②のSL再設計**、サイズ縮小は二次防壁。
